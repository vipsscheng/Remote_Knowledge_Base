# 任务管理系统使用说明

## 简介

本目录包含一个简单的任务管理系统，使用TODO.md文件来跟踪和管理项目任务。通过此系统，您可以：

- 记录和组织项目任务
- 跟踪任务状态（待办、进行中、已完成）
- 管理任务优先级和截止日期
- 监控项目进度

## TODO.md 文件结构

TODO.md文件包含以下几个主要部分：

1. **项目信息**：项目名称、开始日期、预计完成日期和当前状态
2. **任务列表**：按状态分类的任务（待办、进行中、已完成）
3. **任务状态更新流程**：任务状态变更的标准流程
4. **示例任务**：CUDA编程和前端开发的示例任务
5. **项目进度跟踪**：总体进度、最近更新和阻碍/风险
6. **备注**：任务管理的最佳实践

## 任务管理流程

### 1. 添加任务

在"待办任务"部分添加新任务，包括：
- 任务ID（格式：T001、T002等）
- 任务描述
- 优先级（高/中/低）
- 负责人
- 截止日期

### 2. 开始任务

当开始处理一个任务时：
- 将任务从"待办任务"列表移动到"进行中任务"列表
- 更新状态为"进行中"

### 3. 完成任务

当任务完成时：
- 将任务从"进行中任务"列表移动到"已完成任务"列表
- 更新状态为"已完成"
- 将"截止日期"列改为"完成日期"，并填写实际完成日期

### 4. 定期更新

- 定期检查和更新任务状态
- 记录项目中的阻碍和风险
- 更新项目进度信息

## 任务优先级定义

- **高**：关键任务，需要立即处理
- **中**：重要任务，需要在合理时间内完成
- **低**：次要任务，可以在其他任务完成后处理

## 最佳实践

1. **定期更新**：每天或每周更新任务状态，确保任务列表反映真实项目进度
2. **详细描述**：任务描述应清晰明确，避免歧义
3. **合理规划**：根据项目需求和资源情况，合理安排任务优先级和截止日期
4. **及时记录**：遇到阻碍或风险时，及时记录并寻求解决方案
5. **总结反思**：项目结束后，对任务列表进行总结，分析项目执行情况和经验教训

## 工具支持

本目录提供了以下工具来辅助任务管理：

1. **task-manager.py**：Python脚本，用于自动管理TODO.md文件
2. **task-manager.bat**：批处理脚本，提供基本的任务管理功能

### 使用方法

#### Python脚本

```bash
# 显示帮助信息
python task-manager.py help

# 添加任务
python task-manager.py add "实现CUDA内核" "高" "开发者" "2026-03-10"

# 开始任务
python task-manager.py start T001

# 完成任务
python task-manager.py complete T001

# 列出所有任务
python task-manager.py list

# 显示任务状态统计
python task-manager.py status
```

#### 批处理脚本

```bash
# 显示帮助信息
.	ask-manager.bat help

# 添加任务
.	ask-manager.bat add "实现CUDA内核" "高" "开发者" "2026-03-10"

# 开始任务
.	ask-manager.bat start T001

# 完成任务
.	ask-manager.bat complete T001

# 列出所有任务
.	ask-manager.bat list

# 显示任务状态统计
.	ask-manager.bat status
```

## 注意事项

- 确保TODO.md文件使用UTF-8编码，以支持中文
- 任务ID应保持唯一，按顺序递增
- 定期备份TODO.md文件，以防数据丢失
- 团队协作时，应建立统一的任务管理规范
